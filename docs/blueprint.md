# **App Name**: DPX Digital

## Core Features:

- Digital Enrollment & Approval Workflow: Digital forms for participant enrollment with automated approval workflow for admins and HR, including notifications.
- Interactive Agenda & Digital Diary: Allows participants and mentors to create agendas and log activities with multimedia support (text, photos, video).
- AI-assisted Report Generation: Generates a semi-automatic A3 report by summarizing diary entries and feedbacks, using Gemini as a tool for content drafting.
- Feedback Collection from Receiving Unit: Guided form for feedback from receiving units with score and attachments.
- Executive Dashboard for ROI/KPIs: Dashboard for monitoring participant status, tasks, replication efforts, and business/people/culture metrics.
- LGPD Compliance via Digital Signature: Digital signature of terms with consent control to ensure LGPD compliance.
- Role-Based Access Control (RBAC): Configurable access based on roles such as participant, mentor, receiving unit, HR/Committee, and admin.

## Style Guidelines:

- Primary color: Deep blue (#293B5F) for a professional and trustworthy feel, referencing the importance of data security and integrity. This color conveys stability and reliability.
- Background color: Light gray (#E5E8ED), desaturated from the primary color, providing a neutral backdrop that ensures readability and focuses user attention on the content.
- Accent color: Muted teal (#6A9993), an analogous hue to the primary, that contrasts with the blue to highlight important UI elements such as CTAs without overpowering the primary color.
- Body and headline font: 'Inter' for a modern, neutral look suitable for both headlines and body text.
- Use a consistent set of icons from a library like Material Icons or Font Awesome, styled in the accent color (#6A9993), to represent different actions and data types.
- Use a clean, card-based layout with ample white space to present information clearly and to improve the user experience. Focus on a responsive design that adapts to different screen sizes.
- Subtle animations, like fading in new content or transitions between sections, should be used sparingly to provide feedback to the user and make the interface feel more alive, without being distracting.
